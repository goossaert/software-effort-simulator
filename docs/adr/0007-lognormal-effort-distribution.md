# Lognormal effort distribution, parameterised per t-shirt size

Each **Epic** sampled in an **Iteration** has its effort drawn from a *lognormal* distribution `Lognormal(μ, σ)` whose parameters are keyed by the Epic's **T-shirt size**. The synthetic parameters per size are fit so that `P(X < min) ≈ 0.10` and `P(X < max) ≈ 0.90` for the documented P10/P90 band of that size — i.e. `μ = (ln min + ln max) / 2`, `σ = (ln max − ln min) / (2 · Φ⁻¹(0.9))`, with `Φ⁻¹(0.9) ≈ 1.28155`. The **Empirical parameters** mode keeps σ identical but shifts μ by `ln(avg_ratio)` per size, where `avg_ratio` is the mean of `actual / synthetic_mean` over realised Q1 epics.

We chose a lognormal because real engineering effort is strictly positive, right-skewed, and occasionally heavy-tailed — exactly the shape of `exp(N(μ, σ²))`. A normal distribution would assign positive density to negative effort, a triangular distribution would force a hard upper bound that doesn't exist in practice (every engineer has seen an XS turn into a multi-week saga), and a Beta would force an arbitrary rescaling that obscures the per-size meaning. Lognormal also composes nicely with bootstrap resampling of sizes (see [ADR-0006](./0006-monte-carlo-with-bootstrapped-sizes.md)): the per-iteration total is a Poisson-count sum of lognormals, which Monte Carlo can handle directly while no clean closed form exists.

We chose to parameterise *per t-shirt size* rather than fit a single global distribution because the uncertainty itself varies by size: an XL is qualitatively more uncertain than a 2XS — a 2XS that slips to a week is a project, an XL that slips by a week is a normal Tuesday. A single global distribution would smear those regimes together. The cost is a small, hand-maintained `T_SHIRT_PARAMS` table (`index.html:1229`) that has to be updated when sizing conventions change; we accept that because the table is short, lives next to its callers in the single-file app ([ADR-0001](./0001-single-file-html-app.md)), and is also exposed in the sidebar's t-shirt size reference panel so users can see what they're sampling against.

A future revision could fit lognormal parameters directly from empirical data (rather than the current "shift μ by `ln(avg_ratio)`" approximation), or introduce per-team / per-Key-Result parameter sets. We will not move to a different family without first checking that the proposed family still admits the bootstrap-sized Monte Carlo flow described in ADR-0006.
